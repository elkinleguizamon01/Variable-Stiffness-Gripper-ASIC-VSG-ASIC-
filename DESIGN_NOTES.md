# Design Notes: Digital Control System for Variable-Stiffness Gripper

## Overview

This project implements a complete digital control system for a 2-axis robotic arm with a servo-controlled gripper, synthesized in Verilog HDL and targeting the SkyWater SKY130 130nm process through Tiny Tapeout.

## Architecture

### Top-Level Module: `tt_um_brazo_digital`

Serves as the Tiny Tapeout wrapper that translates the standard TT interface into domain-specific signals.

**Port Mapping:**
- `ui_in[7:0]` (inputs): Six command lines from physical buttons/switches
- `uo_out[7:0]` (outputs): Six stepper motor control signals (STEP, DIR for three motors)
- `uio[7:0]` (bidirectional): Unused (tied to 0)
- `clk`: 50 MHz system clock (PIN_R8)
- `rst_n`: Active-low system reset

### Core Module: `brazo_digital_top`

Central hub that orchestrates the entire system:

1. **Input Debouncing Stage** (6× debouncer instances)
   - Cleans physical button/switch noise
   - 20 ms settling time window
   - 2-stage synchronizer prevents metastability

2. **Motor Control Stage** (3× controller instances)
   - X-axis: `nema_controller` (NEMA 17)
   - Y-axis: `nema_controller` (NEMA 17)
   - Gripper: `gripper_stepper_controller` (closed-loop with stiffness feedback)

### Support Modules

#### 1. `debouncer` (Parametric)

**Purpose:** Filter mechanical switch bounce

**Parameters:**
- `STABLE_TIME`: Default = 1,000,000 cycles (20 ms @ 50 MHz)

**Functionality:**
- Synchronizes external asynchronous inputs (2-stage sync)
- Waits for signal to settle before accepting state change
- Prevents glitches from triggering false commands

**Key Design Notes:**
- Synchronizer prevents metastability violations on asynchronous inputs
- Counter accumulates only during transitions
- Resets counter when signal stabilizes

#### 2. `nema_controller`

**Purpose:** Manual direction control for stepper motors

**Inputs:**
- `btn_cw`: Clockwise button
- `btn_ccw`: Counter-clockwise button

**Outputs:**
- `step_pin`: Pulse train to motor driver
- `dir_pin`: Direction flag (High = CW, Low = CCW)

**Step Rate:** 1 kHz (generated by 6,250-cycle frequency divider @ 50 MHz)

**Functionality:**
- Button precedence: If both pressed, motor stops
- Square-wave STEP generation (50% duty cycle)
- Synchronous state machine

#### 3. `gripper_stepper_controller`

**Purpose:** Intelligent gripper with object detection and position tracking

**States:**
- `STATE_OPEN`: Home position, motor stopped
- `STATE_CLOSING`: Active closure, tracking steps taken
- `STATE_HOLD`: Object detected, motor locked at position
- `STATE_OPENING`: Backtracking to home position

**Safety Features:**
- Hard limit: MAX_STEPS = 50,000 (prevents mechanical damage)
- Automatic hold when contact detected (limit switch)
- Position memory: Exact backtrack to home position
- User interrupt: Can abort mid-closing and back out immediately

**Step Rate:** 1 kHz (same as NEMA controller)

**Key Design Notes:**
- Current position counter tracks forward motion
- Decrements during opening to ensure exact return
- Limit switch acts as object detection sensor
- DIP switch controls open/close cycle

## Pin Configuration

### Input Pins (ui_in[7:0])

| Pin | Name | Description |
|-----|------|-------------|
| ui[0] | BTN_X_CW | Button to move X-axis clockwise |
| ui[1] | BTN_X_CCW | Button to move X-axis counter-clockwise |
| ui[2] | BTN_Y_CW | Button to move Y-axis clockwise |
| ui[3] | BTN_Y_CCW | Button to move Y-axis counter-clockwise |
| ui[4] | DIP_GRIP | DIP switch: 1 = close gripper, 0 = release |
| ui[5] | SW_LIMIT | Limit switch: detects gripper contact/stiffness |
| ui[6] | Unused | — |
| ui[7] | Unused | — |

### Output Pins (uo_out[7:0])

| Pin | Name | Description |
|-----|------|-------------|
| uo[0] | STEP_X | Step pulse to DRV8825 X-axis driver |
| uo[1] | DIR_X | Direction signal for X-axis |
| uo[2] | STEP_Y | Step pulse to DRV8825 Y-axis driver |
| uo[3] | DIR_Y | Direction signal for Y-axis |
| uo[4] | STEP_GRIP | Step pulse to gripper motor driver |
| uo[5] | DIR_GRIP | Direction signal for gripper |
| uo[6] | GND | Tied low |
| uo[7] | GND | Tied low |

## Timing Specifications

- **Clock Frequency:** 50 MHz (CLOCK_PERIOD = 20 ns)
- **Step Rate:** 1 kHz (6,250 clock cycles per step transition)
- **Debounce Window:** 20 ms (1,000,000 clock cycles)
- **Step Pulse Width:** Symmetric 50% duty cycle

## Integration with External Hardware

### Stepper Motor Drivers (X & Y Axes)

| Signal | Driver Pin | Notes |
|--------|-----------|-------|
| STEP | STEP | Pull-high required; each rising edge = 1 step |
| DIR | DIR | High = CW; Low = CCW |
| GND | GND | Common ground |

**Recommended Driver Chips:** DRV8825, A4988, ULN2003

### Gripper Motor Driver

Same interface as X/Y axes but with direction controlled by FSM state, not direct button input.

### Limit Switch (Contact Detection)

- **Type:** Normally-open mechanical switch
- **Connection:** Active-high (connected between ui[5] and VCC through 10 kΩ pull-down)
- **Function:** Triggers gripper hold when gripper contacts object

## Testing Strategy

### Gate-Level Simulation

Post-synthesis verification with extracted netlist.

## Performance Notes

- **Area:** Minimal (~2 kGates equivalent)
- **Power:** Dominated by clock distribution (no data-dependent switching)
- **Latency:** ~1 µs from button press to motor step (2-3 synchronization stages)

## Future Enhancements

1. **Configurable Step Rate:** Make frequency divider parametric
2. **PWM Servo Control:** Replace GPIO with 50 Hz PWM for servo gripper
3. **Feedback Encoding:** Integrate encoder reading for position verification
4. **Multi-Speed Modes:** DIP-switch selectable step rates
5. **Error Logging:** SRAM-based event history

## Known Limitations

- Buttons processed every clock cycle (50 ns latency, acceptable for mechanical inputs)
- No velocity ramping (sudden start/stop)
- Single DIP switch for gripper (binary on/off only)
- No LED feedback indicators
